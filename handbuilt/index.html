How it works

The four questions Fleet answers.

Anyone can demo an agent. Running a standing team means knowing who's doing the work, whether it's really happening, what keeps it in bounds, and how to change course — answered here in the same terminal you'd use.

01

Who does the work?

The team

Standing, on your task queue.
$ fleet engagement status eng_4c19
checkout-fixes · active · working
NAME   ROLE       NOW
wren   builder    task #214 · PR #412 open
juno   reviewer   reviewing PR #412
mara   scoper     idle

PR #409 · checks green · merged

A team that stays on. Agents claim tasks from your queue, open pull requests, and merge when CI is green — around the clock.

02

Is it really working?

Live status

Verified, never assumed.
$ fleet engagement status
checkout-fixes   active · working     3 agents on tasks
docs-refresh     active · idle        no agents running
api-hardening    paused

status vs. reality: match · 0 flags

Idle shows as idle. Fleet checks each team's reported status against what its agents are actually doing, and flags any mismatch.

03

What keeps it in bounds?

The rails

You set them at launch.
$ fleet scope show scp_9a41
team        3 agents · 2 builders + reviewer
merge       on green CI  or: hold for your review
budget      $300 / week · hard cap
escalation  blocked → you get pinged

The team works inside your rails. Choose how code merges, cap what a week can cost, and decide when a human gets pulled in — before the first task starts.

04

How do you change course?

Chat

Steer from your terminal.
$ fleet chat send eng_4c19 "invoice fix first"
delivered → checkout-fixes

$ fleet chat follow eng_4c19
wren  moving invoice fix to the top of the queue
wren  PR #418 open · checks running

One message changes the plan. Talk to the team from your terminal; it reorders the queue, answers back, and keeps working.
